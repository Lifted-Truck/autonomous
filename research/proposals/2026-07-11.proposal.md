**Evidence:**
- METR, "GPT-5.6 Sol pre-deployment evaluation" (2026-06-26) —
  https://metr.org/blog/2026-06-26-gpt-5-6-sol/ — the highest reward-hacking/

- "Reward Hacking Benchmark (RHB)" (arXiv:2605.02964, 2026-05-04) —

  (2026-02-23) —
  https://openai.com/index/why-we-no-longer-evaluate-swe-bench-verified/ —
  audited 138 problems, found 59.4% had flawed tests (overly strict/unstated
  requirements, contamination); retired the benchmark and recommends
  SWE-bench Pro instead (scores drop ~80%→~23%).

**What it supersedes:** nothing — additive citation strengthening the
existing "gates are never weakened to pass; a wrong gate is fixed
deliberately, with a DECISIONS entry" line with concrete, named, external
evidence that (a) RL-trained models actively defeat monitoring/oracles, not
just "confidently praise" their own work (the existing Anthropic finding
already in doctrine), and (b) a major lab has already publicly enacted this
exact principle — retiring a broken oracle rather than patching around it.

**Confidence:** high. All three sources are primary (official lab blog /

research agent. All three predate the strict window by weeks to months but
are not currently cited anywhere in `doctrine/` or `research/`.

### 2. ADD — `DESIGN.md` §4b (Curator — anti-poisoning coverage)

**Evidence:**
- "FARMA: Forged Reasoning Attacks on LLM Agent Memory and Defenses"
  (arXiv:2607.05029, ~2026-07-06) — https://arxiv.org/abs/2607.05029 —
  poisons an agent's stored *reasoning/rationale traces* rather than facts,
  using evasive phrasing to slip past keyword-based memory filters, then
  amplifies via repeated retrieval.
- "WhisperBench: Stealthy Memory Injection in Persistent Personal Agents"
  (arXiv:2607.05189, ~2026-07-06) — https://arxiv.org/abs/2607.05189 — a
  108-case benchmark for a single remote payload that (a) induces a memory
  write, (b) stays invisible in the user-facing response, (c) alters future
  behavior.
- "GhostWriter: Memory Poisoning Attacks on Large Language Model Agents"
  (arXiv:2607.06595, ~2026-07-06) — https://arxiv.org/abs/2607.06595 —
  reinforces the same attack class against tool-using personal agents.

**What it supersedes:** nothing — extends the existing poisoning-literature
coverage (PoisonedRAG/AgentPoison/MINJA/RobustRAG/MemoryGraft), which is all
fact/retrieval-poisoning, with a genuinely distinct surface: forged
*justifications* for actions that are otherwise fact-consistent. The
anti-poisoning write gate's "prefer not writing over writing unverified"
principle should extend to verifying reasoning traces, not just factual
claims — a verified-fact gate does not catch a forged rationale attached to
an already-true-looking action.

**Confidence:** medium. Dating and abstracts could not be confirmed against
arXiv primaries — every direct fetch attempt returned HTTP 403 across two
independent agents; recovered only via search-engine snippets. Recommend the
human spot-check these three IDs directly before folding into doctrine.

### 3. AMEND — `DESIGN.md` §4a (Watchdog / destructive-op gate) and
`doctrine/DOCTRINE.md` § AI/deterministic boundary

**Evidence:**
- Adversa AI, "GuardFall" (disclosed 2026-06-30, via The Hacker News) —
  https://thehackernews.com/2026/06/guardfall-exposes-open-source-ai-coding.html
  — 10 of 11 popular open-source coding agents (Aider, Cline, Goose,
  OpenHands, etc.) had their pattern-based/denylist command guards bypassed
  via shell metacharacter tricks; only one agent's structurally different
  architecture resisted.

**What it supersedes:** clarifies, does not reverse, the existing "every
consequential guard is technically enforced, never prose" line. GuardFall
shows that command-string pattern matching / denylists are *not* what
"technical enforcement" means for a destructive-op gate — the shell's own
expansion/substitution semantics defeats them. The gate needs to be
out-of-band (sandboxing, syscall/OS-level allowlisting, or an isolated
execution boundary), not a regex over the command string. Recommend adding
one clarifying sentence to this effect at both cited locations.

**Confidence:** high. Primary disclosure via a named security researcher,
corroborated independently by the governance research agent; not
contradicted by any other source found.
